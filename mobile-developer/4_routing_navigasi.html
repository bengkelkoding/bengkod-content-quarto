4  Routing dan Navigasi

Membuat aplikasi yang lebih dari satu halaman dan menavigasikannya.

4.1 Widget Navigator

Flutter menyediakan kelas Navigator yang bertanggung jawab untuk mengelola tumpukan tampilan. Setiap kali pengguna berpindah ke tampilan baru, Navigator menambahkannya ke tumpukan dan menampilkan tampilan tersebut di layar. Ketika pengguna menekan tombol “kembali” atau melakukan tindakan kembali lainnya, Navigator akan menghapus tampilan teratas dari tumpukan dan menampilkan tampilan sebelumnya.

Ada beberapa jenis navigator yang dapat digunakan dalam Flutter, di antaranya:

4.1.1 MaterialPageRoute

MaterialPageRoute: Ini adalah jenis navigator yang umum digunakan dalam aplikasi Flutter yang mengikuti desain material dari Flutter. MaterialPageRoute menampilkan transisi animasi bawaan saat tampilan baru ditampilkan atau ditutup.

4.1.2 CupertinoPageRoute

CupertinoPageRoute: Ini adalah jenis navigator yang mengikuti desain iOS dengan menggunakan transisi animasi khas iOS. Jika Anda ingin aplikasi Anda memiliki tampilan yang mirip dengan aplikasi iOS, Anda dapat menggunakan CupertinoPageRoute.

4.2 Navigasi Antar Halaman

Ada beberapa metode navigasi yang umum digunakan dalam Flutter:

4.2.1 push

push: Metode ini menambahkan rute baru ke tumpukan dan membawa pengguna ke halaman baru. Contoh kode navigator membawa pengguna dari halaman pertama ke halaman kedua.

ElevatedButton(
    child: const Text('Go to Second Screen'),
    onPressed: () {
        Navigator.push(context,
            MaterialPageRoute(builder: (context) => MySecondPage()));
    },
),

4.2.2 pop

pop: Metode ini menghapus rute saat ini dari tumpukan dan membawa pengguna kembali ke halaman sebelumnya. Contoh navigator menghapus halaman kedua dan membawa pengguna kembali ke halaman pertama.

ElevatedButton(
    child: const Text('Back to First Screen'),
    onPressed: () {
        Navigator.pop(context);
    },
),

4.2.3 pushReplacement

pushReplacement: Metode ini mengganti rute saat ini dengan rute baru. Ini berguna ketika Anda ingin mengganti halaman tetapi tidak ingin menjaga halaman sebelumnya di tumpukan. Contoh navigator menggantikan halaman saat ini dengan halaman baru.

ElevatedButton(
    child: const Text('Back to First Screen'),
    onPressed: () {
        Navigator.pushReplacement(context, MaterialPageRoute(builder: (context) => MySecondPage()));
    },
),

4.3 Navigasi antar halaman dengan nama route

Ada beberapa metode navigasi dengan nama route yang digunakan dalam Flutter:

4.3.1 pushNamed

pushNamed: Metode ini menambahkan rute baru ke tumpukan dan membawa pengguna ke halaman baru dengan nama route.

Berikut adalah contoh penggunaan Navigator.pushNamed() di dalam Flutter untuk melakukan navigasi dengan nama routes:

Pertama, tentukan nama routes di dalam MaterialApp:

MaterialApp(
  title: 'MyApp',
  initialRoute: '/',
  routes: {
    '/': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

Kemudian, untuk melakukan navigasi ke halaman dengan nama route, gunakan method Navigator.pushNamed() dan berikan nama route yang ingin dituju:

Navigator.pushNamed(context, '/detail');

4.3.2 popAndPushNamed

popAndPushNamed: Metode ini digunakan untuk menghapus halaman saat ini dari tumpukan dan menambahkan halaman baru dengan nama route.

Berikut adalah contoh penggunaan Navigator.popAndPushNamed() di dalam Flutter untuk menghapus halaman saat ini dari tumpukan dan menambahkan halaman baru dengan nama route:

Pertama, tentukan nama routes di dalam MaterialApp:

MaterialApp(
  title: 'MyApp',
  initialRoute: '/',
  routes: {
    '/': (context) => HomePage(),
    '/detail': (context) => DetailPage(),
  },
);

Kemudian, untuk menghapus halaman saat ini dari tumpukan dan menambahkan halaman baru dengan nama route, gunakan method Navigator.popAndPushNamed() dan berikan nama route yang ingin dituju:

Navigator.popAndPushNamed(context, '/result');

4.4 Berbagi Data Antar Halaman

Dalam Flutter, terdapat beberapa cara untuk berbagi data antara halaman atau layar dalam aplikasi:

4.4.1 Menggunakan Konstruktor

Menggunakan Konstruktor: Mengirim data menggunakan konstruktor dilakukan dengan cara menginisialisasi data yang ingin dikirim pada konstruktor halaman yang ingin dituju.

// Halaman Sumber
Navigator.push(
    context,
    MaterialPageRoute(
    builder: (context) => HalamanTujuan(
        data: dataYangDibagikan,
        ),
    ),
);

// Halaman Tujuan
class HalamanTujuan extends StatelessWidget {
    final String data;

    HalamanTujuan({required this.data});

    // ...
}

4.4.2 Menggunakan ModalRoute

Menggunakan ModalRoute: Untuk mengambil data kembali dari halaman tujuan ke halaman sumber setelah halaman tujuan ditutup, dapat menggunakan ModalRoute dalam kombinasi dengan metode pop.

// Halaman Sumber
final data = await Navigator.push(
    context,
    MaterialPageRoute(builder: (context) => HalamanTujuan()),
);
// Proses data yang dikembalikan dari Halaman Tujuan

// Halaman Tujuan
class HalamanTujuan extends StatelessWidget {
    // ...
    // Pada saat ingin mengirimkan data kembali ke halaman sumber
    Navigator.pop(context, dataYangDikirimkanKembali);
    // ...
}

4.4.3 Menggunakan State Management

Menggunakan State Management: Jika data yang perlu dibagikan di seluruh aplikasi berjumlah banyak, dapat menggunakan paket state management seperti provider, riverpod, maupun bloc untuk mengelola state aplikasi secara global.

4.5 State Management

Seperti yang sudah dijelaskan sebelumnya jika memiliki data yang banyak keberadaan state management menjadi diperlukan. Ada banyak jenis state management namun pada dasarnya mereka semua sama yaitu untuk membantu perpindahan data dalam flutter.

Pada modul ini state management yang akan digunakan merupakan flutter_riverpod. Flutter Riverpod merupakan pustaka dalam Flutter yang digunakan untuk manajemen state dan dependency injection. Riverpod didasarkan pada konsep “provider” dan memberikan solusi yang lebih sederhana dan fleksibel untuk manajemen state dibandingkan dengan paket Flutter lainnya seperti Provider atau GetX.

Untuk menggunakan flutter_riverpod terlebih dahulu tambahkan pustakanya didalam pubspec.yaml lalu ketik perintah flutter pub get di terminal atau dengan menggunakan perintah flutter pub add flutter_riverpod. Lalu setelah itu deklarasikan scope dari provider dengan cara sebagai berikut :

void main() {
    runApp(const ProviderScope(child: MainApp()));
}

Paket flutter_provider menyediakan berbagai macam provider yang digunakan untuk manajemen state dan dependency injection dalam aplikasi Flutter. Berikut adalah provider-provider yang umum digunakan dalam flutter_provider:

  1. Provider: Provider ini digunakan untuk menyediakan objek tanpa adanya state yang dapat berubah. Berguna untuk menyediakan ketergantungan yang tidak memerlukan pembaruan atau perubahan state.

  2. StateProvider: Provider ini digunakan untuk menyediakan objek yang nilainya dapat dimodifikasi dari luar. Berguna dalam kondisi sederhana seperti melakukan fiter atau memberikan data yang bisa diakses dan diperbarui dari berbagai halaman.

  3. FutureProvider: Provider ini digunakan untuk mengelola hasil dari Future. Ini memungkinkan Anda untuk melakukan pemrosesan asinkron dan mengubah UI ketika future selesai atau mengalami perubahan.

  4. StateNotifierProvider: Provider ini merupakan provider khusus yang harus digunakan bersama dengan state notifier. State notifer merupakan sebuah kelas yang menyimpan satu tipe data untuk dan dapat diamati setiap aktvitasnya oleh StateNotifierProvider. Provider beserta kelas tersebut dapat digunakan untuk memanipulasi state tingkat lanjut, yang akan sulit direpresentasikan dengan provider yang lebih sederhana seperti [Provider] atau [FutureProvider].
